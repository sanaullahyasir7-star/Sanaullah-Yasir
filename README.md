### 🧠 Engineering Philosophy
* **Evaluation-Driven Design:** If you can't measure it, you can't improve it. I build automated evaluation pipelines (using RAGAS, TruLens) before optimizing prompts or architectures.
* **Cost-Aware Inference:** I don't just build for accuracy; I build for the bottom line. I optimize token usage, implement semantic caching, and right-size model deployments.
* **Software Engineering First:** AI code is still software. I write modular, tested, and documented code, and I treat my ML pipelines with the same CI/CD rigor as backend services.

### 🚀 Featured Projects

Here are a few systems I've architected and deployed end-to-end:

#### 🏢 [Enterprise RAG Pipeline with Hybrid Search]
* **The Problem:** Internal company search was failing due to keyword mismatch and context loss.
* **The Solution:** Built a production RAG system using a hybrid search approach (BM25 + Dense Vector) with `pgvector`. Implemented advanced chunking strategies and a re-ranking layer using Cross-Encoders.
* **The Impact:** Reduced search hallucination rate by 40% and cut average query latency to < 800ms.
* 🔗 **[[https://github.com/sanaullahyasir7-star/-Automated-LLM-Fine-Tuning-Serving-Pipeline.](https://github.com/sanaullahyasir7-star/enterprise-rag-pipeline)]**

#### ⚙️ [Automated LLM Fine-Tuning & Serving Pipeline]
* **The Problem:** Fine-tuning open-source models (like Llama 3 / Mistral) was too manual and slow for the data science team.
* **The Solution:** Architected an end-to-end MLOps pipeline. Automated data preprocessing, QLoRA fine-tuning via Hugging Face TRL, and automated deployment to a vLLM inference server on AWS ECS.
* **The Impact:** Reduced model iteration time from 2 weeks to 2 days, and cut inference costs by 30% compared to using proprietary APIs.
* 🔗 **[[Link to Repo / Case Study]([https://github.com/sanaullahyasir7-star/-Automated-LLM-Fine-Tuning-Serving-Pipeline.](https://github.com/sanaullahyasir7-star/-Automated-LLM-Fine-Tuning-Serving-Pipeline.))]**

#### 🤖 [Autonomous Data Analysis Agent]
* **The Problem:** Non-technical stakeholders needed a way to query complex SQL databases using natural language.
* **The Solution:** Developed a ReAct-based AI agent using LangGraph. The agent can dynamically generate SQL, execute it safely in a sandboxed environment, and generate matplotlib/plotly visualizations based on the results.
* **The Impact:** Empowered 50+ non-technical users to self-serve data insights, saving the analytics team ~10 hours a week.
* 🔗 **[[Link to Repo / Case Study](https://github.com/sanaullahyasir7-star/Autonomous-Data-Analysis-Agent)]**
